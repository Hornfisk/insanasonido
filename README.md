# InsanaSonido

Event promotion site for underground electronic music on Spain's Costa del Sol — Techno, Tech-House, Tech-Trance, Hardgroove, Makina. Presented by Sonido Sessions & Project Insana.

**Live site:** [insanasonido.es](https://insanasonido.es)

---

## What this is

A fast, SEO-optimised static site built to promote underground electronic music events in Benalmádena, Málaga. The design language — black, magenta glow, CRT noise overlay, glitch animations — is intentional: it mirrors the aesthetic of the scene it represents.

Built with a deliberate bias towards simplicity: no JS framework, no CSS utility library, no bloat. The entire site loads in under a second.

---

## Tech stack

| Layer | Choice | Why |
|---|---|---|
| Framework | [Astro 6](https://astro.build) | Zero-JS by default, file-based routing, content collections, native i18n |
| Styling | Vanilla CSS + custom properties | Full control, no build step complexity, no class soup |
| CMS | [Decap CMS](https://decapcms.org) at `/420/` | Git-based, no database, works with Netlify Identity |
| Hosting | [Netlify](https://netlify.com) | Edge functions for geo-redirect, custom security headers |
| Fonts | Google Fonts — Space Grotesk + Inter | Display + body pairing, preconnected for performance |
| Analytics | GA4 with consent gate | Fires only after user consent; consent.js + analytics.js |

**Dependencies:** `astro`, `@astrojs/sitemap` — that's it.

## SEO implementation

This is where most of the deliberate engineering lives. Rather than relying on a plugin to handle everything, the structured data and meta layer is hand-rolled in `Base.astro` to give full control per page type:

- **`MusicEvent` JSON-LD** — event pages: name, date, venue, organizers, performers, ticket tiers with `Offer` schema
- **`BlogPosting` JSON-LD** — blog posts: headline, published date, publisher
- **`Organization` + `WebSite` JSON-LD** — all non-event pages
- **`BreadcrumbList` JSON-LD** — blog posts and event pages, rendered visually too
- **Open Graph + Twitter Card** — all pages, locale-aware (`og:locale`)
- **`hreflang`** — ES/EN alternates + `x-default` on every page, derived programmatically from `canonicalPath`
- **Canonical URLs** — explicit on every page to prevent duplicate content between language versions
- **XML sitemap** — auto-generated by `@astrojs/sitemap` with i18n config

## Internationalisation

Astro's built-in i18n with `prefixDefaultLocale: false` — Spanish is the default locale (no `/es/` prefix), English lives under `/en/`. All content is duplicated in both languages via the content collection structure. The `useTranslations()` helper pulls from typed dictionaries in `src/i18n/`.

---

## Security

All headers are set in `netlify.toml` — no runtime overhead:

- `Content-Security-Policy` — strict allowlist; relaxed only under `/420/` for the CMS
- `Strict-Transport-Security` with `preload`
- `X-Content-Type-Options`, `X-Frame-Options`, `Referrer-Policy`, `Permissions-Policy`
- CMS route (`/420/`) has `X-Robots-Tag: noindex, nofollow`

## Getting started

Requires Node ≥ 22.12.0.

```sh
npm install
npm run dev        # http://localhost:4321
npm run build      # production build to ./dist/
npm run preview    # preview the build locally
```

## Adding content

### New event

1. Create `src/content/events/es/your-slug.md` and `src/content/events/en/your-slug.md`
2. Follow the schema in `src/content.config.ts` — include `urlSlug`, `coverImage`, `ticketTiers`, `performers`, etc.
3. Create the event page at `src/pages/[slug]/index.astro` and `src/pages/en/[slug]/index.astro`
4. Pass `breadcrumbs`, `eventSchema`, and `logoHref` to `Base.astro` and `Header.astro`

### New blog post

1. Create `src/content/blog/es/your-post.md` and `src/content/blog/en/your-post.md`
2. Dynamic routes (`src/pages/blog/[...slug].astro`) pick them up automatically
3. Pass `blogSchema` and `breadcrumbs` to `Base.astro` in the page template

## Design system

| Token | Value |
|---|---|
| Background | `#000000` |
| Accent | `#D900FF` (magenta) |
| Text | `#FFFFFF` |
| Display font | Space Grotesk 400/700/900 |
| Body font | Inter 300/400/600 |

Atmosphere is built from three CSS layers: `bg-gradient` (radial magenta glow), `grid-overlay` (perspective grid), and `#crt` (animated noise texture via SVG filter). All defined in `global.css`.

---

## Deployment

Pushes to `main` deploy automatically to Netlify. A geo-redirect edge function (`netlify/edge-functions/geo-redirect.js`) handles locale routing based on visitor country.
